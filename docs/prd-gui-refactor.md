# GUI 重构：合并三种工作流到单页面界面

## Problem Statement

当前 MinerU GUI 存在两个独立的功能模式（MinerU 转换和翻译），用户需要在不同模式之间切换才能使用完整功能。同时，翻译功能依赖外部项目 `fast_pdf_trans`，需要将其整合到 MinerU 项目中。

用户希望：
1. 在单一页面中使用所有功能，无需切换模式
2. 支持三种工作流：PDF 转 Markdown、Markdown 翻译、完整翻译管道
3. 通过 GUI 配置 LLM 设置，无需编辑配置文件

## Solution

重构 GUI 为单页面设计，支持三种工作流的统一入口：
1. **工作流 1（完整翻译管道）**：PDF → Markdown → 翻译 → PDF
2. **工作流 2（Markdown 翻译）**：Markdown → 翻译 → PDF/Markdown
3. **工作流 3（PDF 转 Markdown）**：PDF → Markdown

同时将 `fast_pdf_trans` 的翻译模块迁移到 MinerU 项目中，实现功能整合。

## User Stories

1. 作为用户，我想要在单一界面中选择三种工作流之一，以便快速开始所需的任务
2. 作为用户，我想要通过 GUI 配置 LLM 设置（API 地址、模型名称、API 密钥），以便管理翻译功能
3. 作为用户，我想要在工作流 2 中自动查找关联的 PDF 文件，以便进行标题校正
4. 作为用户，我想要在工作流 1 中指定页码范围，以便只处理部分页面
5. 作为用户，我想要选择标题校正方式（无/书签/LLM/LLM 简化版），以便获得正确的标题层级
6. 作为用户，我想要在工作流 2 中选择是否生成 PDF，以便灵活控制输出格式
7. 作为用户，我想要选择 GPU 或 CPU 模式，以便根据硬件条件优化性能
8. 作为用户，我想要在运行过程中查看实时日志，以便监控任务进度
9. 作为用户，我想要在任务运行时取消操作，以便在出错时及时停止
10. 作为用户，LLM 配置应该持久化到配置文件，以便下次启动时保留设置
11. 作为用户，首次启动时应读取现有配置文件，以便无缝迁移现有设置

### 核心模块

1. **LLM 配置管理模块** (`llm_translate/config.py`)
   - 加载/保存 conf.yaml
   - 提供 LLM 配置读写接口
   - GUI 调用此模块读写配置

2. **翻译器模块** (`llm_translate/translator.py`)
   - `translate(md_path)` - 翻译 markdown
   - 复用现有 LLM 调用逻辑

3. **标题校正模块** (`llm_translate/corrector/`)
   - `HeaderCorrector` - LLM 标题校正
   - `EasyHeaderCorrector` - 简化版标题校正
   - `HeaderFactory` - 工厂类

4. **GUI 主模块** (`gui.py`)
   - 工作流切换逻辑
   - 选项显示/隐藏
   - PDF 自动查找逻辑
   - 进程管理

### 工作流逻辑

**工作流 1（PDF → 翻译 PDF）**：
1. MinerU 转换 PDF → Markdown
2. 标题校正（使用 PDF 书签或 LLM）
3. LLM 翻译 Markdown
4. 生成翻译后 PDF

**工作流 2（Markdown → 翻译 PDF）**：
1. 自动在 markdown 同目录查找同名 PDF
2. 如未找到，弹框让用户选择（可跳过）
3. LLM 翻译 Markdown
4. 标题校正
5. （可选）生成 PDF

**工作流 3（PDF → Markdown）**：
1. MinerU 转换 PDF → Markdown

### LLM 配置对话框

弹窗包含：
- API 地址（base_url）
- 模型名称（model_name）
- API 密钥（api_key）

保存到项目根目录 `conf.yaml`。

### 配置加载逻辑

1. 首次启动时，检查 `conf.yaml` 是否存在
2. 如存在，读取现有配置
3. 如不存在，使用硬编码默认值
4. GUI 修改配置后，保存到 `conf.yaml`

## Testing Decisions

### 测试策略

1. **单元测试**：针对核心模块（配置管理、翻译器、标题校正）
2. **集成测试**：模拟完整工作流执行
3. **GUI 测试**：验证界面交互逻辑

### 测试重点

- 配置加载/保存正确性
- 工作流命令构建正确性
- PDF 自动查找逻辑
- 选项显示/隐藏逻辑

## Out of Scope

- CLI 翻译命令开发（仅 GUI）
- 多 LLM Agent 配置
- 高级 LLM 配置（超时、重试、流控等）
- 其他文件格式支持（仅 PDF 和 Markdown）
